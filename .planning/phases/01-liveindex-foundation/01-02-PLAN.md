  **src/live_index/store.rs — Core data structures:**

  `ParseStatus` enum: `Parsed`, `PartialParse { warning: String }`, `Failed { error: String }`. Per CONTEXT.md this is a queryable field.

  `IndexedFile` struct: `relative_path: String`, `language: LanguageId`, `content: Vec<u8>` (LIDX-03), `symbols: Vec<SymbolRecord>` (LIDX-02), `parse_status: ParseStatus`, `byte_len: u64`, `content_hash: String`.
  Impl `IndexedFile::from_parse_result(result: FileProcessingResult, bytes: Vec<u8>) -> Self` — maps `FileOutcome` to `ParseStatus`, stores content bytes regardless of parse outcome (total-failure files store content per user decision "leaning toward store content").

  `CircuitBreakerState` struct: `total: AtomicUsize`, `failed: AtomicUsize`, `tripped: AtomicBool`, `threshold: f64`, plus `Vec<(String, String)>` for first 5 failure details (path + reason) behind a `Mutex`. Constructor takes threshold param (default from `TOKENIZOR_CB_THRESHOLD` env var, fallback 0.20).
  Methods: `record_success()`, `record_failure(path: &str, reason: &str)`, `should_abort() -> bool` (returns false if total < 5 — minimum file threshold per research pitfall #3), `is_tripped() -> bool`, `summary() -> String` (one-line summary + top 3-5 failures per CONTEXT.md).

  `IndexState` enum: `Loading`, `Ready`, `CircuitBreakerTripped { summary: String }`.

  `LiveIndex` struct: `files: HashMap<String, IndexedFile>`, `loaded_at: std::time::Instant`, `load_duration: std::time::Duration`, `cb_state: CircuitBreakerState`.
  `pub type SharedIndex = Arc<RwLock<LiveIndex>>;`

  `LiveIndex::load(root: &Path) -> crate::error::Result<SharedIndex>`:
  1. Call `discovery::discover_files(root)?`
  2. Read threshold from `TOKENIZOR_CB_THRESHOLD` env var (parse as f64, default 0.20)
  3. Use `rayon::prelude::*` — `discovered.par_iter().map(|df| { read bytes, call parsing::process_file, return (path, IndexedFile) }).collect::<Vec<_>>()`
  4. Build HashMap sequentially from results, calling `cb_state.record_success()` or `cb_state.record_failure()` for each entry. If `cb_state.should_abort()` returns true, break and log circuit breaker trip via `tracing::error!`.
  5. Record `loaded_at` and `load_duration`.
  6. Wrap in `Arc::new(RwLock::new(index))` and return.

  CRITICAL: `LiveIndex::load()` is SYNC (not async). It runs before the tokio runtime starts. Rayon handles parallelism internally. Per research: "Perform the entire LiveIndex::load() synchronously before the tokio MCP server starts."

  CRITICAL: Never use `println!` or `print!`. All logging via `tracing::info!`, `tracing::warn!`, `tracing::error!`.

  **src/live_index/query.rs — Query methods on &LiveIndex (NOT &SharedIndex):**

  Per research pitfall #4: "Structure query methods to accept `&LiveIndex` (after the guard is taken by the caller) rather than taking `&SharedIndex`." This prevents re-entrant RwLock deadlocks.

  `HealthStats` struct: `file_count: usize`, `symbol_count: usize`, `parsed_count: usize`, `partial_parse_count: usize`, `failed_count: usize`, `load_duration: std::time::Duration`. Derive `Debug, Clone`.

  Implement these methods on `impl LiveIndex`:

  - `pub fn get_file(&self, relative_path: &str) -> Option<&IndexedFile>` — O(1) HashMap lookup (LIDX-02)
  - `pub fn symbols_for_file(&self, relative_path: &str) -> &[SymbolRecord]` — returns symbols slice or empty slice if file not found
  - `pub fn all_files(&self) -> impl Iterator<Item = (&String, &IndexedFile)>` — iterate all entries
  - `pub fn file_count(&self) -> usize` — `self.files.len()`
  - `pub fn symbol_count(&self) -> usize` — sum of symbols.len() across all files
  - `pub fn is_ready(&self) -> bool` — `!self.cb_state.is_tripped()`
  - `pub fn index_state(&self) -> IndexState` — returns Ready or CircuitBreakerTripped based on cb_state
  - `pub fn health_stats(&self) -> HealthStats` — iterates files to compute parsed/partial/failed counts, plus file_count, symbol_count, load_duration. Per CONTEXT.md: "Health stats include: file counts, symbol counts, parse status breakdown, total index load duration."

  Note: These methods take `&self` (i.e., `&LiveIndex`). Callers must first acquire the RwLock read guard: `let index = shared_index.read().unwrap(); index.get_file("path")`. This pattern is enforced by the type system — `SharedIndex` is `Arc<RwLock<LiveIndex>>`, not a wrapper type.
