<task type="auto">
  <name>Task 2: Wire token savings from sidecar /stats into MCP health tool</name>
  <files>src/protocol/format.rs, src/protocol/tools.rs</files>
  <action>
    **In src/protocol/format.rs:**
    1. Add a new public function `pub fn format_token_savings(stats_json: &str) -> String` that parses the StatsSnapshot JSON from the sidecar /stats endpoint and formats a "Token Savings" section:
       ```
       ── Token Savings (this session) ──
       Read:  N fires, ~M tokens saved
       Edit:  N fires, ~M tokens saved
       Write: N fires
       Grep:  N fires, ~M tokens saved
       Total: ~T tokens saved
       ```
       If the stats JSON is empty or unparseable, return an empty string (fail-open — the health report still works without savings).

    2. Add unit tests for `format_token_savings` with known StatsSnapshot JSON input.

    **In src/protocol/tools.rs:**
    3. In the `health` tool handler, after computing the existing health report:
       - Read the sidecar port from `.tokenizor/sidecar.port` (same pattern as hook.rs `read_port_file`).
       - Make a sync HTTP GET to `127.0.0.1:{port}/stats` with a 50ms timeout. Use the same raw TcpStream approach as hook.rs or a simpler `reqwest::blocking` if available. Since the health tool runs inside the async MCP server, use `tokio::task::spawn_blocking` to avoid blocking the tokio executor during the sync HTTP call.
       - Parse the response body, call `format::format_token_savings(&body)`.
       - Append the formatted section to the health report string.
       - If the sidecar is unreachable (port file missing, connection refused), skip the savings section silently.

    4. **Alternative simpler approach:** Since the MCP server and sidecar run in the same process and share the same `Arc<SharedIndex>`, consider passing `Arc<TokenStats>` into `TokenizorServer` via a new field. This avoids the HTTP round-trip entirely. The TokenStats is created in `main.rs` when spawning the sidecar, and the same Arc is passed to `TokenizorServer::new()`.
       - This requires adding a `token_stats: Option<Arc<TokenStats>>` field to `TokenizorServer`.
       - In the health handler: `if let Some(ref stats) = self.token_stats { let snap = stats.summary(); format::format_token_savings_from_snapshot(&snap) }`.
       - This approach is simpler, faster, and avoids the sidecar HTTP call. Choose this approach per Claude's Discretion from CONTEXT.md.

    5. Update `TokenizorServer::new()` in `src/protocol/mod.rs` to accept an optional `Arc<TokenStats>`.
    6. Update `main.rs` to pass the `Arc<TokenStats>` from the sidecar's `SidecarState` into `TokenizorServer::new()`.

    **Files also touched (minor):**
    - `src/protocol/mod.rs`: Add `token_stats` field to `TokenizorServer`, update `new()` constructor.
    - `src/main.rs`: Pass `token_stats` Arc from sidecar state to MCP server.
