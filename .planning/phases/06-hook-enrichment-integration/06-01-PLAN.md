<objective>
Augment all sidecar handlers to return enriched, human-readable, budget-enforced text responses instead of raw JSON arrays. Add TokenStats atomic counters for per-session token savings tracking, a SidecarState struct that bundles index + stats + symbol cache, and a /stats endpoint.

Purpose: The sidecar is the data source for all hook responses. Enriching handlers server-side means the hook binary just formats the text into additionalContext JSON — no heavy processing in the sync hook binary. Budget enforcement server-side guarantees hook output stays within token limits.

Output: All 5 existing handlers return formatted text, /stats endpoint added, TokenStats recording in every handler, symbol cache for impact diff.
</objective>
