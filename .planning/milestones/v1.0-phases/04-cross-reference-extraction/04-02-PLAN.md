    Add to src/live_index/query.rs:

    1. Built-in type filter constants — hardcoded per-language lists per user decision:
    ```rust
    const RUST_BUILTINS: &[&str] = &["i8","i16","i32","i64","i128","isize","u8","u16","u32","u64","u128","usize","f32","f64","bool","char","str","String","Self","self"];
    const PYTHON_BUILTINS: &[&str] = &["int","float","str","bool","list","dict","tuple","set","None","bytes","object","type"];
    const JS_BUILTINS: &[&str] = &["string","number","boolean","undefined","null","Object","Array","Function","Symbol","Promise","Error"];
    const TS_BUILTINS: &[&str] = &["string","number","boolean","undefined","null","void","never","any","unknown","Object","Array","Function","Symbol","Promise","Error","Record","Partial","Required","Readonly","Pick","Omit"];
    const GO_BUILTINS: &[&str] = &["int","int8","int16","int32","int64","uint","uint8","uint16","uint32","uint64","float32","float64","complex64","complex128","bool","string","byte","rune","error","any"];
    const JAVA_BUILTINS: &[&str] = &["int","long","short","byte","float","double","boolean","char","void","String","Object","Integer","Long","Short","Byte","Float","Double","Boolean","Character"];
    ```

    2. Single-letter generic filter:
    ```rust
    const SINGLE_LETTER_GENERICS: &[&str] = &["T","K","V","E","R","S","A","B","C","D","N","M","P","U","W","X","Y","Z"];
    ```

    3. `fn is_filtered_name(name: &str) -> bool` — returns true if name appears in any language's built-in list OR in single-letter generics. This is a coarse filter applied at query time.

    4. Query method:
    ```rust
    pub fn find_references_for_name(
        &self,
        name: &str,
        kind_filter: Option<ReferenceKind>,
        include_filtered: bool, // false = apply built-in/generic filters
    ) -> Vec<(&str, &ReferenceRecord)>
    ```
    - Look up `name` in `self.reverse_index`
    - For each ReferenceLocation, resolve to the actual ReferenceRecord via `self.files[file_path].references[reference_idx]`
    - Apply kind_filter if Some
    - Apply built-in/generic filter if !include_filtered and name matches
    - Also check all files' alias_maps: if any file has alias_map entry (alias -> original) where original == name, also look up the alias in the reverse_index. This implements XREF-05.
    - If name contains "::" or ".", treat as qualified: match against qualified_name field. Otherwise match against name field.
    - Return Vec of (file_path, &ReferenceRecord) tuples

    5. Dependents query:
    ```rust
    pub fn find_dependents_for_file(&self, target_path: &str) -> Vec<(&str, &ReferenceRecord)>
    ```
    - Iterate all files. For each file, check its references where kind == Import.
    - An import reference is a dependent if the import's name (or qualified_name) text is a suffix/segment match of target_path. Heuristic: check if the import text contains the file stem (e.g., import text "db" or "crate::db" matches target "src/db.rs").
    - Return Vec of (importing_file_path, &import_reference)

    6. Helper for get_context_bundle (Plan 03 will consume):
    ```rust
    pub fn callees_for_symbol(
        &self,
        file_path: &str,
        symbol_index: usize,
    ) -> Vec<&ReferenceRecord>
    ```
    - Returns references in the given file whose enclosing_symbol_index == symbol_index and kind == Call.
    - These are the "callees" — functions called BY the target symbol.
