    Create src/parsing/xref.rs with:

    1. Per-language query strings as `const &str`:
       - RUST_XREF_QUERY: call_expression (identifier, scoped_identifier, field_expression), macro_invocation, use_declaration (with use_as_clause for alias), type_identifier. Add `(call_expression function: (scoped_identifier) @ref.qualified_call)` for qualified names per Pitfall 7.
       - PYTHON_XREF_QUERY: call (identifier, attribute), import_statement, import_from_statement, type annotations (type (identifier))
       - JS_XREF_QUERY: call_expression (identifier, member_expression), import_statement, import_specifier
       - TS_XREF_QUERY: same as JS plus type_identifier for TypeScript type references
       - GO_XREF_QUERY: call_expression (identifier, selector_expression), import_spec, type_identifier
       - JAVA_XREF_QUERY: method_invocation (identifier), import_declaration, type_identifier

       Use the node type tables from docs/summaries/research-xref-extraction-and-file-watching.md as starting reference. Test and iterate query strings against actual tree-sitter grammar nodes.

    2. OnceLock<Query> statics per language (6 total). A helper function per language:
       ```rust
       static RUST_QUERY: OnceLock<Query> = OnceLock::new();
       fn rust_query(lang: &tree_sitter::Language) -> &'static Query {
           RUST_QUERY.get_or_init(|| Query::new(lang, RUST_XREF_QUERY).expect("valid rust xref query"))
       }
       ```

    3. Main extraction function:
       ```rust
       pub fn extract_references(
           root: &tree_sitter::Node,
           source: &str,
           language: &LanguageId,
       ) -> (Vec<ReferenceRecord>, HashMap<String, String>)
       ```
       - Get the compiled Query for the language
       - Use QueryCursor to iterate matches
       - Map capture names to ReferenceKind:
         - "ref.call" / "ref.method_call" -> Call
         - "ref.qualified_call" -> Call with qualified_name set to full node text, name set to last segment
         - "ref.import" -> Import
         - "ref.type" -> TypeUsage
         - "ref.macro" -> MacroUse
         - "import.original" + "import.alias" -> populate alias_map
       - Build Vec<ReferenceRecord> and HashMap<String,String> alias_map
       - For unsupported languages, return (vec![], HashMap::new())

    4. Wire into src/parsing/mod.rs:
       - Add `pub mod xref;` to src/parsing/mod.rs (alongside existing `pub mod languages;`)
       - In parse_source, after `languages::extract_symbols`, call `xref::extract_references(&root, source, language)` to get (references, alias_map)
       - Return the extended tuple: `Ok((symbols, has_error, references, alias_map))`
       - In process_file, propagate references and alias_map into FileProcessingResult fields

    Note: The enclosing_symbol_index is NOT set during extraction. It is set later in IndexedFile::from_parse_result (Task 1) where both symbols and references are available.

    Add `#[cfg(test)] mod tests` inside xref.rs with per-language unit tests.
