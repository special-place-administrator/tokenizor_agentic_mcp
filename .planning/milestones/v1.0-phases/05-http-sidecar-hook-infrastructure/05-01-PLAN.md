Uses `std::net::TcpStream::connect_timeout`.
   - Unit tests: test_write_read_port_roundtrip, test_cleanup_removes_files, test_read_port_missing_returns_error

  IMPORTANT: All file paths use `.tokenizor/` relative to current working directory. The sidecar.port file contains ONLY the port number as ASCII digits, no newline. This is the convention the hook binary (Plan 02) will rely on.

<task type="auto">
  <name>Task 2: Sidecar router, endpoint handlers, and spawn function</name>
  <files>src/sidecar/router.rs, src/sidecar/handlers.rs, src/sidecar/server.rs</files>
  <action>
1. Create src/sidecar/handlers.rs with 5 GET endpoint handlers. All handlers follow this pattern:
   - Accept `State(index): State&lt;SharedIndex&gt;` and `Query(params): Query&lt;ParamsStruct&gt;` (where applicable)
   - Acquire index.read() guard, extract owned data, drop guard, return Json response
   - On poisoned lock, return StatusCode::INTERNAL_SERVER_ERROR
   - On file not found, return StatusCode::NOT_FOUND
   - Define request/response structs with Deserialize/Serialize derives

   Handlers:
   a. `health_handler(State(index))` — returns JSON with file_count, symbol_count, index_state, uptime. No query params.
   b. `outline_handler(State(index), Query(OutlineParams { path }))` — returns JSON array of symbols for the given file path (name, kind, start_line, end_line). If file not in index, 404.
   c. `impact_handler(State(index), Query(ImpactParams { path }))` — calls find_dependents_for_file on the path; returns JSON array of {file, references: [{line, name}]}. If file not in index, 404.
   d. `symbol_context_handler(State(index), Query(SymbolContextParams { name, file: Option }))` — calls find_references_for_name(name, None); returns JSON array of {file, references: [{line, kind, enclosing}]}. Cap at 50 results total.
   e. `repo_map_handler(State(index))` — returns JSON array of {path, symbol_count, parse_status} for all files. No query params.

   Each handler has at least 1 unit test (mock a SharedIndex with test data, call handler directly).

2. Create src/sidecar/router.rs:
   - `pub fn build_router(index: SharedIndex) -> axum::Router` — creates Router with 5 GET routes:
     - GET /health -> health_handler
     - GET /outline -> outline_handler
     - GET /impact -> impact_handler
     - GET /symbol-context -> symbol_context_handler
     - GET /repo-map -> repo_map_handler
   - Uses `.with_state(index)` to inject SharedIndex

3. Create src/sidecar/server.rs:
   - `pub async fn spawn_sidecar(index: SharedIndex, bind_host: &str) -> anyhow::Result&lt;SidecarHandle&gt;`
   - Calls `check_stale(bind_host)` to clean up any stale files
   - Binds `TcpListener::bind(format!("{bind_host}:0"))`
   - Gets port via `listener.local_addr()?.port()`
   - Calls `port_file::write_port_file(port)` and `port_file::write_pid_file(std::process::id())`
   - Creates oneshot channel for shutdown
   - Calls `router::build_router(index)` to get the Router
   - `tokio::spawn` the axum::serve with graceful_shutdown wired to the oneshot receiver
   - Inside the spawned task, after serve completes, call `port_file::cleanup_files()`
   - Return `SidecarHandle { port, shutdown_tx }`
   - Log at tracing::info level: "sidecar listening on {bind_host}:{port}"

  IMPORTANT anti-patterns to avoid:
  - Do NOT hold RwLockReadGuard across any .await point in handlers — extract data into owned Vec/String, drop guard, then return
  - Do NOT write anything to stdout — all tracing goes to stderr via existing tracing infrastructure
  - Bind host default is "127.0.0.1", configurable via TOKENIZOR_SIDECAR_BIND env var (read in spawn_sidecar)
  </action>
