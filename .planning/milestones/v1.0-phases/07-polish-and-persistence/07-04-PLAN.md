<objective>
Add symbol extraction, cross-reference queries, and tree-sitter parsing for 8 additional languages:
C#, Ruby, PHP, Swift, Kotlin, Dart, Perl, and Elixir. This brings total language support from 8 to 16,
covering the vast majority of popular programming languages.
</objective>

<context>
The architecture is already prepared — `LanguageId` enum has variants for C#, Ruby, PHP, Swift, Dart, Perl, Elixir
(Kotlin needs to be added). Extension mappings in `from_extension()` and `extensions()` already exist for all
except Kotlin. Each language follows an identical integration pattern established by the existing 8 languages.

**Pattern per language (4 touchpoints):**
1. `Cargo.toml` — add `tree-sitter-{lang}` dependency
2. `src/parsing/languages/{lang}.rs` — `extract_symbols()` with walk_node + find_name
3. `src/parsing/mod.rs` — match arm in `parse_source()` for grammar selection
4. `src/parsing/xref.rs` — xref query string + OnceLock + match arm in `extract_references()`

Plus shared wiring: `languages/mod.rs` match arm, `domain/index.rs` support_tier update, tests.

**Crate versions (all verified available):**
- tree-sitter-c-sharp = "0.23.1"
- tree-sitter-ruby = "0.23.1"
- tree-sitter-php = "0.24.2"
- tree-sitter-swift = "0.7.1"
- tree-sitter-kotlin = "0.3.8"
- tree-sitter-dart = "0.0.4"
- tree-sitter-perl = "1.1.2"
- tree-sitter-elixir = "0.3.5"

**IMPORTANT implementation notes:**
- tree-sitter-php exposes two grammars: `LANGUAGE_PHP` (full with `<?php` tags) and `LANGUAGE_PHP_ONLY` (code only). Use `LANGUAGE_PHP` for the parser and xref since real PHP files have tags.
- tree-sitter-swift and tree-sitter-kotlin may use different API patterns (some expose `LANGUAGE` constant, others use `language()` function). Check the crate docs.
- tree-sitter-dart v0.0.4 may have limited node types — keep symbol extraction conservative.
- Kotlin needs a new `LanguageId::Kotlin` variant added to the enum with extension "kt"/"kts".
</context>

**domain/index.rs — add Kotlin variant:**
- Add `Kotlin` to `LanguageId` enum
- Add `"kt" | "kts" => Some(Self::Kotlin)` to `from_extension()`
- Add `Self::Kotlin => &["kt", "kts"]` to `extensions()`
- Add Kotlin to `Display` impl
- Update `support_tier()`: move C#, Ruby, PHP, Swift, Kotlin, Dart, Perl, Elixir from `Unsupported` to `Broader`

**Create 8 language files in src/parsing/languages/:**
Each follows the same pattern as python.rs (simplest template):

- **csharp.rs**: node kinds — `class_declaration`, `struct_declaration`, `interface_declaration`, `enum_declaration`, `method_declaration`, `constructor_declaration`, `namespace_declaration`, `property_declaration`. Name child: `name` (identifier).
- **ruby.rs**: node kinds — `method` (Function), `class` (Class), `module` (Module), `singleton_method` (Method). Name child: `name` (identifier).
- **php.rs**: node kinds — `function_definition`, `class_declaration`, `interface_declaration`, `trait_declaration`, `method_declaration`, `enum_declaration`. Name child: `name` (identifier).
- **swift.rs**: node kinds — `function_declaration`, `class_declaration`, `struct_declaration`, `enum_declaration`, `protocol_declaration`. Name child: `name` (simple_identifier).
- **kotlin.rs**: node kinds — `function_declaration`, `class_declaration`, `object_declaration`, `interface_declaration`. Name child: `simple_identifier`.
- **dart.rs**: node kinds — `function_signature` (parent: top-level), `class_definition`, `enum_declaration`, `method_signature`. Name child: `identifier`.
- **perl.rs**: node kinds — `subroutine_declaration_statement` (Function), `package_statement` (Module). Name child: `name` or `identifier`.
- **elixir.rs**: node kinds — `call` with target `def`/`defp`/`defmodule`. Elixir's tree-sitter grammar models functions as calls to `def`. Use a pattern that checks call target.

**Wire into languages/mod.rs:**
Add `mod` declarations and match arms for all 8 languages.

**Wire into parsing/mod.rs:**
Add match arms in `parse_source()` for grammar selection. Check each crate's API for how the language constant is exposed:
- Most use `LANGUAGE` constant (e.g., `tree_sitter_ruby::LANGUAGE`)
- Some may use `language()` function — check and adapt
- PHP: use `tree_sitter_php::LANGUAGE_PHP`

**Tests: verify symbol extraction works for each language.**
Add basic tests in the existing test infrastructure — at minimum, one test per language that parses a trivial source and verifies at least one symbol is extracted.

<task id="2" name="Add cross-reference queries for 8 new languages" commit_message="feat(07): add xref queries for C#, Ruby, PHP, Swift, Kotlin, Dart, Perl, Elixir">

**xref.rs — add 8 query strings + OnceLock statics + accessor functions + match arms:**

Query patterns per language (covering calls, imports, types):

- **C#**: method invocations, object creation (`new`), using directives, type identifiers
- **Ruby**: method calls (identifier + `.method`), `require`/`require_relative`, constant references
- **PHP**: function calls, method calls (`->method()`), `use` statements, `new` expressions, type identifiers
- **Swift**: function calls, method calls (`.method()`), `import` declarations, type identifiers
- **Kotlin**: function calls, method calls (`.method()`), `import` directives, type identifiers
- **Dart**: function calls, method invocations, `import` directives, type identifiers
- **Perl**: subroutine calls, `use`/`require` statements
- **Elixir**: function calls (Module.function), `alias`/`import`/`use` directives

For each language:
1. Add a `const {LANG}_XREF_QUERY: &str` with tree-sitter query patterns
2. Add a `static {LANG}_QUERY: OnceLock<Query>`
3. Add an accessor function `fn {lang}_query(lang: &Language) -> &'static Query`
4. Add match arm in `extract_references()`
5. Add match arm in test helper `parse_and_extract()`

**Add xref tests:**
- At least one test per language verifying a call reference is extracted
- At least one test per language verifying an import reference is extracted
- Add all 8 languages to the `test_all_languages_produce_at_least_one_ref` and `test_empty_source_produces_empty_refs` cross-language tests
