All functions take `&LiveIndex` (or relevant data extracted from it) and return `String`. No I/O, no async.

**Functions to implement, matching CONTEXT.md locked formats exactly:**

1. `pub fn file_outline(index: &LiveIndex, path: &str) -> String`
   - Header: `{path}  ({N} symbols)`
   - Body: for each symbol, indent by `depth * 2` spaces, then `{kind_prefix:<12} {name:<30} {start_line}-{end_line}` (right-aligned line range). Kind prefix comes from SymbolKind Display.
   - Not-found: "File not found: {path}" (no crash)

2. `pub fn symbol_detail(index: &LiveIndex, path: &str, name: &str, kind_filter: Option<&str>) -> String`
   - Returns full source body (slice `IndexedFile.content` by `byte_range`) using `String::from_utf8_lossy`
   - Footer: `[{kind}, lines {start}-{end}, {byte_count} bytes]`
   - Not-found: "No symbol {name} in {path}. Symbols in that file: {comma-separated list}" or "File not found: {path}"
   - If kind_filter provided, filter symbols by kind Display match (case-insensitive)

3. `pub fn search_symbols_result(index: &LiveIndex, query: &str) -> String`
   - Case-insensitive substring match on symbol names across all files
   - Header: `{N} matches in {M} files`
   - Body: grouped by file path, each match shows `  {line_start}: {kind} {name}`
   - Empty: "No symbols matching '{query}'"

4. `pub fn search_text_result(index: &LiveIndex, query: &str) -> String`
   - Case-insensitive substring match on file content lines
   - Header: `{N} matches in {M} files`
   - Body: grouped by file, each match shows `  {line_number}: {line_content}`
   - Handle CRLF by trimming \r from line ends
   - Empty: "No matches for '{query}'"

5. `pub fn repo_outline(index: &LiveIndex) -> String`
   - Header: `{project_name}  ({N} files, {M} symbols)` where project_name is derived from root dir name or "project"
   - Body: directory tree using `  ` indentation, each file shows `{filename:<20} {language:<12} {symbol_count} symbols`
   - Sort by path for deterministic output

6. `pub fn health_report(index: &LiveIndex) -> String`
   - Format:
     ```
     Status: {Ready|Empty|Degraded}
     Files:  {N} indexed ({P} parsed, {PP} partial, {F} failed)
     Symbols: {S}
     Loaded in: {D}ms
     Watcher: not active (Phase 3)
     ```

7. `pub fn what_changed_result(index: &LiveIndex, since_ts: i64) -> String`
   - Compare `since_ts` (Unix seconds) against `loaded_at_system`
   - If since < loaded_at: return list of all files (entire index is "newer")
   - If since >= loaded_at: return "No changes detected since last index load."

8. `pub fn file_content(index: &LiveIndex, path: &str, start_line: Option<u32>, end_line: Option<u32>) -> String`
   - Return raw content from memory, optionally sliced by 1-indexed line range
   - Not-found: "File not found: {path}"

9. `pub fn not_found_file(path: &str) -> String` — "File not found: {path}"
10. `pub fn not_found_symbol(index: &LiveIndex, path: &str, name: &str) -> String` — "No symbol {name} in {path}. Symbols in that file: ..."
11. `pub fn loading_guard_message() -> String` — "Index is loading... try again shortly."
12. `pub fn empty_guard_message() -> String` — "Index not loaded. Call index_folder to index a directory."

Write comprehensive unit tests in a `#[cfg(test)] mod tests` block within format.rs. Tests should construct LiveIndex instances directly (using the existing test helper patterns from store.rs/query.rs) and verify exact output format.
