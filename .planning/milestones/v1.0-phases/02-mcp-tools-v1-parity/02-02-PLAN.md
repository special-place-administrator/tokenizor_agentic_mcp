**Tool handlers** — each follows the pattern:
1. Acquire read lock (or write lock for index_folder)
2. Check loading guard (except health which always responds)
3. Extract data from LiveIndex into owned values
4. Drop lock
5. Call format:: function
6. Return String

**The 10 tools with their descriptions (model-facing text):**

1. `get_file_outline` — "Return the symbol outline for a file. Shows functions, structs, classes with line ranges."
2. `get_symbol` — "Look up a specific symbol by file path and name. Returns full source code."
3. `get_symbols` — "Batch lookup of symbols or code slices. Each target can be a symbol name or byte range."
4. `get_repo_outline` — "Show the file tree with language and symbol counts per file."
5. `search_symbols` — "Search for symbols by name substring across all indexed files."
6. `search_text` — "Full-text search across all indexed file contents."
7. `health` — "Report server health: index status, file counts, load duration." (NO loading guard)
8. `index_folder` — "Reload the index from a directory path. Replaces current index entirely."
9. `what_changed` — "Show files changed since a Unix timestamp."
10. `get_file_content` — "Serve file content from memory with optional line range."

**Loading guard pattern** (applied to all except health):
```rust
let guard = self.index.read().expect("lock poisoned");
match guard.index_state() {
    IndexState::Ready => {}
    IndexState::Empty => return format::empty_guard_message(),
    IndexState::Loading => return format::loading_guard_message(),
    IndexState::CircuitBreakerTripped { summary } => {
        return format!("Index degraded: {summary}");
    }
}
// Extract needed data into owned values here, then:
drop(guard);
```

**index_folder special handling:**
- Acquires WRITE lock: `let mut guard = self.index.write().expect("lock poisoned");`
- Calls `guard.reload(&PathBuf::from(&params.0.path))?` — if error, return error message as String
- After reload, format a success message with file/symbol counts

**get_symbols batch handling:**
- For each target in `params.0.targets`:
  - If `name` is Some: call `format::symbol_detail()`
  - If `start_byte`/`end_byte` are Some: slice `file.content[start..end]`, return with path header
  - Concatenate all results separated by `\n---\n`

**Anti-patterns to avoid (per RESEARCH.md):**
- Do NOT return JSON from any tool handler — always plain text String
- Do NOT use MCP error codes for not-found — return helpful text via format functions
- Do NOT hold RwLockReadGuard across await points — extract into owned values first, then drop
- Do NOT add JsonSchema derive to domain types (SymbolRecord, IndexedFile) — only on input param structs

Write unit tests verifying:
- Loading guard returns correct message for Empty/Loading/CB states
- Tool handlers with mock index return expected format (at least 2-3 representative tools)
