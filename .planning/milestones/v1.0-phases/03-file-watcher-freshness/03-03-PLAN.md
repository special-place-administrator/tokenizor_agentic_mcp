  <name>Task 2: Integration tests for all FRSH requirements + RELY-03</name>
  <files>tests/watcher_integration.rs</files>
  <action>
    Create `tests/watcher_integration.rs` with integration tests that exercise the watcher end-to-end using real filesystem operations and the watcher module.

    Test setup pattern for each test:
    ```rust
    use std::sync::{Arc, RwLock, Mutex};
    use std::time::Duration;
    use tempfile::TempDir;
    use tokenizor_agentic_mcp::live_index::LiveIndex;
    use tokenizor_agentic_mcp::watcher::{WatcherInfo, run_watcher};

    // 1. Create a tempdir with a .rs source file
    // 2. Load LiveIndex from the tempdir
    // 3. Create WatcherInfo with default()
    // 4. Spawn run_watcher as a tokio task
    // 5. Wait ~100ms for watcher to initialize
    // 6. Perform filesystem operation (create/modify/delete)
    // 7. Wait ~500ms for watcher to process (debounce 200ms + processing margin)
    // 8. Assert LiveIndex state reflects the change
    // 9. Drop the tempdir (watcher task will error and exit)
    ```

    Tests to create (use `#[tokio::test]` attribute):

    1. **test_watcher_detects_modify_and_updates_index** (FRSH-01, FRSH-03, FRSH-06):
       - Create tempdir with `src/hello.rs` containing `fn hello() {}`
       - Load index, spawn watcher
       - Overwrite `src/hello.rs` with `fn hello_world() {}`
       - Wait 500ms
       - Assert: `index.read().get_file("src/hello.rs")` symbols contain "hello_world", NOT "hello"

    2. **test_watcher_indexes_new_file** (FRSH-04):
       - Create tempdir with `src/existing.rs`
       - Load index, spawn watcher
       - Create NEW file `src/new_file.rs` containing `fn new_function() {}`
       - Wait 500ms
       - Assert: `index.read().get_file("src/new_file.rs")` is Some with symbols containing "new_function"
       - Assert: file_count increased by 1

    3. **test_watcher_removes_deleted_file** (FRSH-05):
       - Create tempdir with `src/to_delete.rs` containing `fn doomed() {}`
       - Load index, spawn watcher, verify file is indexed
       - Delete `src/to_delete.rs`
       - Wait 500ms
       - Assert: `index.read().get_file("src/to_delete.rs")` is None
       - Assert: file_count decreased by 1

    4. **test_watcher_hash_skip_on_noop_write** (content_hash optimization):
       - Create tempdir with `src/stable.rs` containing `fn stable() {}`
       - Load index, spawn watcher, note loaded_at_system
       - Overwrite `src/stable.rs` with SAME content `fn stable() {}`
       - Wait 500ms
       - Assert: symbols unchanged (hash skip worked — no re-parse happened)
       - Note: Cannot directly assert "no re-parse" from outside, but can verify symbols are identical and content_hash matches

    5. **test_watcher_enoent_handled_gracefully** (RELY-03):
       - Create tempdir with `src/fragile.rs`
       - Load index, spawn watcher
       - Delete `src/fragile.rs` (simulates ENOENT scenario)
       - Wait 500ms
       - Assert: file removed from index, NO panic, watcher still running (check watcher_info state is Active, not Degraded)

    6. **test_single_file_reparse_under_50ms** (FRSH-02):
       - This is a performance micro-benchmark, not a watcher test
       - Create a tempdir with a .rs file containing a moderate function (20 lines)
       - Read bytes, call `parsing::process_file` directly
       - Measure elapsed time with Instant::now() + elapsed()
       - Assert: elapsed < Duration::from_millis(50)

    7. **test_watcher_state_reports_active** (health extension):
       - Create tempdir, load index, spawn watcher
       - Wait 200ms for watcher to initialize
       - Assert: watcher_info.lock().state == WatcherState::Active

    8. **test_watcher_ignores_non_source_files**:
       - Create tempdir with `src/code.rs`, load index, spawn watcher
       - Create `README.md` in the tempdir
       - Wait 500ms
       - Assert: `index.read().get_file("README.md")` is None (file not indexed)
       - Assert: file_count unchanged

    **Timing notes:**
    - Use 500ms wait times (200ms debounce + 300ms margin) for reliability
    - Mark tests with `#[ignore]` attribute if they prove flaky on CI (timing-sensitive)
    - Tests should clean up by letting the tempdir drop (watcher will see errors and exit)
    - All tests use `#[tokio::test]` for async runtime
