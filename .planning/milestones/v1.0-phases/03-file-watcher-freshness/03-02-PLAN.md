Output: A complete `src/watcher/mod.rs` with `run_watcher()` async function, `start_watcher()` helper, event processing pipeline, and unit tests for path normalization, hash skip, and ENOENT handling.

       pub(crate) fn normalize_event_path(abs_path: &Path, repo_root: &Path) -> Option<String>

       ```rust
       pub(crate) fn maybe_reindex(
           relative_path: &str,
           abs_path: &Path,
           shared: &SharedIndex,
           language: LanguageId,
       ) -> ReindexResult
       ```
       Where `ReindexResult` is a local enum: `{ HashSkip, Reindexed, Removed, ReadError(String) }`.
       - Read file bytes with std::fs::read(abs_path)
       - On ENOENT: acquire write lock, call remove_file(relative_path), log warn, return Removed
       - On other read error: log warn, return ReadError
       - Compute new_hash via crate::hash::digest_hex(&bytes)
       - Acquire READ lock, check if existing file's content_hash == new_hash. If match, drop lock, log debug "hash-skip", return HashSkip
       - Drop read lock BEFORE parsing (critical: do NOT hold lock during parse)
       - Call parsing::process_file(relative_path, &bytes, language)
       - Call IndexedFile::from_parse_result(result, bytes)
       - Acquire WRITE lock, call update_file(relative_path.to_string(), indexed)
       - Log debug "re-indexed {relative_path}"
       - Return Reindexed

  <action>
    Add the watcher lifecycle functions to `src/watcher/mod.rs`, below the functions from Task 1.

    1. **WatcherHandle struct** (extends the existing type stubs or replaces them):
       ```rust
       pub struct WatcherHandle {
           // The debouncer owns the OS watcher thread; dropping it stops watching
           _debouncer: Debouncer<RecommendedWatcher, RecommendedCache>,
           event_rx: std::sync::mpsc::Receiver<DebounceEventResult>,
       }
       ```

    2. **start_watcher function:**
       ```rust
       pub(crate) fn start_watcher(
           repo_root: &Path,
       ) -> Result<WatcherHandle, notify::Error>
       ```
       - Create std::sync::mpsc::channel for DebounceEventResult
       - Create debouncer via `new_debouncer(Duration::from_millis(200), None, move |result| { let _ = tx.send(result); })`
       - Call `debouncer.watcher().watch(repo_root, RecursiveMode::Recursive)?`
       - Return WatcherHandle with debouncer and rx

    3. **process_events function:**
       ```rust
       pub(crate) fn process_events(
           events: Vec<DebouncedEvent>,
           repo_root: &Path,
           shared: &SharedIndex,
           burst_trackers: &mut HashMap<PathBuf, BurstTracker>,
           watcher_info: &Arc<Mutex<WatcherInfo>>,
       )
       ```
       - For each event in the batch:
         - Skip if !is_relevant_event(&event)
         - For each path in event.paths (DebouncedEvent can have multiple paths):
           - Call normalize_event_path(path, repo_root). Skip if None.
           - Check supported_language(path). Skip if None (not a supported extension).
           - For EventKind::Remove: acquire write lock on shared, call remove_file. Update watcher_info (events_processed++, last_event_at).
           - For EventKind::Create or EventKind::Modify:
             - Update burst_trackers entry for this path. Check effective_debounce_ms.
             - Call maybe_reindex(relative_path, abs_path, shared, language)
             - Update watcher_info (events_processed++, last_event_at, debounce_window_ms from tracker)

    4. **run_watcher async function** (the public entry point):
       ```rust
       pub async fn run_watcher(
           repo_root: PathBuf,
           shared: SharedIndex,
           watcher_info: Arc<Mutex<WatcherInfo>>,
       )
       ```
       - This is the supervision loop spawned by main.rs
       - Set watcher_info state to Active at start
       - Loop with consecutive_failures counter (max 3):
         - Call start_watcher(&repo_root). On error: increment failures, log warn, if >= 3 set state to Degraded and break, else sleep 1s and retry.
         - On success: reset consecutive_failures to 0, set state to Active
         - Enter event processing loop: `while let Ok(result) = handle.event_rx.recv()`
           - On Ok(events): call process_events(...)
           - On Err(errors): log warn for each error, increment a per-session error counter; if too many, break inner loop to trigger restart
         - When event_rx.recv() returns Err (channel closed = watcher dropped): log warn, increment failures, sleep 1s, retry from start_watcher
       - After breaking out of restart loop (degraded), watcher_info state is Degraded. Log error.

    5. **restart_watcher function** (called by index_folder):
       ```rust
       pub fn restart_watcher(
           repo_root: PathBuf,
           shared: SharedIndex,
           watcher_info: Arc<Mutex<WatcherInfo>>,
       ) -> tokio::task::JoinHandle<()>
       ```
       - Sets watcher_info state to Off briefly, then spawns a new `tokio::spawn(run_watcher(...))` task
       - Returns the JoinHandle

    6. Run `cargo test --lib` and `cargo check` to verify compilation.

    NOTE on anti-patterns to avoid:
    - Do NOT use tokio::sync::mpsc in the notify callback — notify's callback runs on its own OS thread, use std::sync::mpsc
    - Do NOT use spawn_blocking — notify runs its own thread
    - Do NOT hold SharedIndex write lock during tree-sitter parse (maybe_reindex already handles this correctly from Task 1)
