## Project Overview

Tokenizor is being developed as a Rust-native, coding-first indexing and retrieval platform. In its current state, the codebase is a foundation slice, while the `docs/` directory defines the intended product direction more completely than the implementation alone.

This documentation set should therefore be read using a dual-state model:

- **Current implemented state:** the Rust scaffold and its demonstrated behavior
- **Intended target state:** the architecture and product direction described in the project documents

## Quick Reference

- **Current stack:** Rust 2024, Tokio, `rmcp`, Serde, tracing
- **Current entry points:** `src/main.rs`, `src/lib.rs`
- **Current external surfaces:** CLI commands plus MCP `health`
- **Current strongest implementation area:** byte-exact local CAS scaffold and deployment/health wiring
- **Target architecture center:** engine first, MCP as universal transport, provider-native adapters later
